# 07 · 里程碑与实现顺序

按"先把核心爽点跑通，再美化、再上桌面"的顺序。每个里程碑可独立验收。

## M0 · 项目骨架

- Vite + React + TS 工程初始化，目录结构按 [04-ARCHITECTURE.md](./04-ARCHITECTURE.md)。
- 接入 Zustand、Zod、idb；配置 Vitest / Playwright。
- 占位 `<GameCanvas>`：PixiJS Application 挂载成功，渲染一块空地图网格。
- **验收**：本地起服务，看到暖色空地图。

## M1 · 领域逻辑（纯函数，TDD）

- `earnings.ts`：年产出 → 每秒速率 `r`。
- `harvest.ts`：收成节流算法（[03 文档](./03-HARVEST-ALGORITHM.md)），含单测：
  不漂移、频率上限、X≥1、跨周期补发。
- `currency.ts`：货币/金币换算与格式化。
- **验收**：领域单测全绿，覆盖率达标。

## M2 · 单资产端到端（核心爽点）

- 添加一个写死的现金流资产 → 地图出现一个（占位）单位。
- 主循环驱动：运作动画 + 进度条爬升 + 满格 `+X` 飘字 + 顶部数字累加。
- IndexedDB 持久化 + 刷新恢复。
- **验收**：能看到一个单位持续产出、`+X` 正常弹、刷新不丢数据。
  —— 这是项目的"魔法时刻"，最早能感受到产品价值。

## M3 · 完整录入与多资产 ✅（2026-06-14 完成）

- ✅ 真实录入：单弹窗表单（类型/名字/外观皮肤/金额），Zod 校验（`domain/assetInput.ts`）。
  - 实现为**单面板弹窗**而非 4 步向导（字段少，单面板更顺手）：`ui/AssetModal`。
  - 货币/隐私是**全局设置**，在 `ui/SettingsModal`（⚙ 设置）里改，不在录入里重复。
- ✅ 外观选择器 `ui/SkinPicker`：卡片展示**皮肤实时缩略图**（`engine/skinThumbnail` 离屏渲染）。
- ✅ 多资产并行运作；**点击单位 → 编辑/删除**（board `setOnUnitTap` → `AssetModal` 预填 + 删除按钮）。
- ✅ 本日收成午夜重置；离线回归补发；localStorage 持久化。
- **验收**：可自由增删改资产、两类资产、隐私模式均正确。已浏览器实测。

## M4 · 美术与动效打磨

- 程序化占位图升级 / 或接入首批 AI 精灵图（待出图通道确定）。
- 收成粒子、数字跳动、环境氛围动画、像素 UI 面板。
- **验收**：第一眼"可爱/治愈"达标。

## M5 · 桌面挂件（Phase 2）

- Tauri v2 集成，复用全部 Web 代码。
- 透明置顶窗口 + 托盘 + 挂件精简布局。
- **验收**：Mac/Windows 桌面常驻挂件可看到资产产出。

---

## ✅ 已拍板的决策

| # | 问题 | 决定 |
|---|---|---|
| 1 | 货币 | **全局单一货币**，不支持多币种；货币与金币比例都是全局设置 |
| 2 | 离线挂机收益 | **要有**（补发；超阈值合并为"离线收成 +N"汇总，避免海量动画） |
| 3 | 切换货币/模式后历史累计 | **不重算**；另提供「清零按钮」让用户主动重新累积 |
