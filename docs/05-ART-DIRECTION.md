# 05 · 美术方向

## 1. 风格基调：星露谷 / 牧场物语 暖色田园

- **关键词**：温暖、治愈、手绘感、柔和、可爱、生机。
- **情绪**：让人放松、想多看一会儿；强调"生长"与"丰收"。
- **像素规格**：单位以 16×16 或 32×32 为基准网格绘制，整数倍缩放显示（保持像素清晰，
  开启 nearest-neighbor 缩放，关闭抗锯齿）。

## 2. 配色（暖色田园基调）

## 3. 资产单位 = 会生产的"小世界"（项目成败所在）

资产不是一个静态图标，而是一个**有细节、有角色、像活着的生产小世界**——
这才是吸引人、产生"我的钱在为我工作"实感的关键。**细节就是产品。**

- 例：现金流"麦田场" = 三排会随风摆的麦子 + **两个干活的农民（锄地、搬运）+ 一个偷懒的农民** + 一袋粮。
- 例：投资"果园/矿井/作坊" = 多个工人各司其职、机器运转、产物堆积。
- 头顶进度条像"成熟度/产出进度"，满了"丰收"：作物闪光、农民欢呼、弹 `+X`。

### 分层原则（给"做细"留空间）

每个资产场景拆成可独立替换/加细的层：
**土壤/场地 → 作物/产物 → 角色（多个，各有行为）→ 道具 → 特效**。
角色再拆成 身体/手臂/工具/表情，每个都能单独加帧、加动作、加随机性（偷懒、打哈欠、擦汗…）。

> 默认皮肤用 Pixi 程序化绘制（可糙），但结构上必须分层，方便后续把任意一层换成
> 更精细的 AI 精灵图 / 手绘帧，而不重写场景逻辑。用户可从预设资产皮肤中选择（添加/编辑时）。

## 4. 动画规范（项目灵魂）

每个单位至少三种状态动画：

1. **运作（idle/working）**：循环播放的微动作（作物轻摇、风车转、家禽踱步）。
   - 2~4 帧，循环周期 0.6~1.2s，错峰播放避免整齐划一的机械感。
2. **收成（harvest）**：进度条满瞬间的"丰收"动作 + 满格闪光。
   - 一次性，~0.3s，可带轻微缩放弹跳（squash & stretch）。
3. **`+X` 飘字**：从单位头顶向上飘 ~24px、同时放大后淡出，~0.8s。
   - 金黄描边，数字越大字号略大（>1 时更醒目，呼应"产出快"的爽感）。

补充氛围动画（加分项，非 MVP 必须）：

- 进度条满时小粒子（金币/星星）迸溅。
- 环境：飘动的云、偶尔飞过的小鸟、昼夜暖光微变化。

## 5. 占位美术策略（MVP，先跑通）

在接入 AI 精灵图之前，用**程序化像素图**保证开发不被美术阻塞：

- 用 Canvas / SVG 在代码里绘制简单像素单位（色块 + 描边 + 少量帧动画）。
- 遵循上面的色板与网格规格，保证后续替换为 AI 图时布局/尺寸不变。
- 每个图标定义为一个 `IconDef`：尺寸、帧数、绘制函数 / 帧数据。
- 接入正式精灵图时，只替换纹理来源，动画时序与挂载逻辑不变。

```ts
interface IconDef {
  id: string;
  size: 16 | 32;
  frames: number;          // 运作动画帧数
  draw?: (ctx, frame) => void;   // 占位：程序化绘制
  atlas?: { url: string; ... };  // 正式：精灵图集
}
```

## 6. AI 精灵图管线（后期接入）

确定方向后切换到 AI 出图（需出图通道，见 README）：

1. 用统一 prompt 模板（风格、色板、视角、网格、留白）批量生成单位精灵 + 帧。
2. 后处理：抠背景、对齐网格、打包成 texture atlas（如 TexturePacker / 免费替代）。
3. 替换 `IconDef.atlas`，下线占位绘制。
4. 沉淀一份"像素资产美术规范" skill（可选），固化 prompt 与后处理流程。

## 7. UI 视觉（HUD）

- 木质/羊皮纸质感的像素面板，圆角描边。
- 字体：像素字体（拉丁 + 中文像素字体；中文像素字体较少，必要时正文用清晰无衬线，
  仅数字/标题用像素字体）。🟡 OPEN：中文像素字体选型。
- 顶部"本日/累积收成"：大号像素数字 + 滚动跳动动画（数字增长时有"跳一下"反馈）。
