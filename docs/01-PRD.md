# 01 · 产品需求文档（PRD）

## 1. 愿景与理念

**目的：** 增强每个人对自己资产的"拥有感"，由拥有感衍生出安全感与幸福感。

**洞察：** 资产对大多数人是抽象的数字（银行 App 里冷冰冰的余额）。当资产被
具象化为"地图上一个个会动、会赚钱的可爱小单位"，并提供**持续、即时的正反馈**
（运作动画 + 进度条 + `+X` 收成），用户与自己财富的情感连接会显著增强。

**类比：** 模拟经营游戏（星露谷、模拟城市、放置类游戏）里看着自己产业增长的爽感，
迁移到"看着真实资产增长"的体验上。

### 设计第一性原则

1. **画面与动画是核心竞争力**，不是装饰。资产运作动画 = 项目灵魂。
2. **即时正反馈**：用户每隔几秒就能看到地图上有东西在动、在产出。
3. **治愈而非焦虑**：暖色、柔和、可爱。强调"增长"，淡化波动与亏损。
4. **隐私友好**：数据本地存储；提供"金币"模式隐藏真实金额。

## 2. 目标用户

- 有一定资产（工资、投资、房租、退休金等）、希望对财富有掌控感的个人。
- 喜欢模拟经营 / 放置类游戏审美的人。
- 注重隐私、不愿把财务数据上传云端的人。

## 3. 平台策略

1. **Phase 1 — Web**：浏览器内完整体验。
2. **Phase 2 — 桌面组件**：Mac/Windows 桌面挂件（Tauri 打包，复用 Web 代码）。
   - 形态设想：可常驻桌面、半透明置顶的小窗口，瞥一眼就能看到资产在产出。

**约束：尽量不重复开发** → 选型必须保证 Web 与桌面共用同一套渲染与逻辑代码。
（见 [04-ARCHITECTURE.md](./04-ARCHITECTURE.md)）

## 4. MVP 范围

### 4.1 地图

- 方格（grid）地图，整体像素风。
- 资产以"小单位"形式落在格子上，持续运作。
- MVP 不需要复杂地形/扩张；先保证"放得下、看得清、动得起来"。

### 4.2 资产类型（两类）

| 类型 | 用户输入 | 收益逻辑 |
|---|---|---|
| **现金流（Cashflow）** | 名字、皮肤、**年收入** | 按年收入匀速产出 |
| **投资（Investment）** | 名字、皮肤、**当前金额**、**预期年收益率** | 按 `金额 × 年收益率` 匀速产出 |

> **类型 ≠ 皮肤**：现金流/投资**只**是录入项与收益算法的差异；外观皮肤是与类型无关的
> 共享池，任何皮肤（麦田场/果园/…，均为"生产小世界"）都能配任何类型。见 [08-SKINS.md](./08-SKINS.md)。

> 两类的收益最终都归一为"每秒产出速率 r"，喂给收成节流算法。
> 见 [03-HARVEST-ALGORITHM.md](./03-HARVEST-ALGORITHM.md)。

### 4.3 录入信息

- **名字**：自由文本（如"我的工资"、"沪深300"）。
- **图标**：即该资产的小单位外观/动画。MVP 提供一组预设像素图标供选择。
- **金额**：现金流填年收入；投资填当前金额 + 预期年收益率。以全局货币录入。

> **货币与隐私模式是全局设置，不在单个资产上录入**（已与用户确认）：
> - **货币**：全局唯一，从 USD / EUR / GBP / JPY / CNY / SGD 中选一个，全图通用。
> - **隐私模式**：全局开关；开启后填一个全局"金币兑换比例"，此后所有展示以金币为单位。
> - 详见 [02-DATA-MODEL.md](./02-DATA-MODEL.md)。

### 4.4 运作与收成（核心体验）

1. 资产落到地图后**持续运作**（运作动画循环播放）。
2. 每个单位头顶/旁边有一条**进度条**。
3. 进度条满 → 触发**收成动画**，头顶弹出 `+X`（X 带单位）。
4. **节流规则（硬约束）：**
   - 进度条"满"的频率**最快 0.5 秒**一次（不会比这更快）。
   - `+X` 中的 X **最小为 1**。
   - 产出快 → 每 ~0.5s 弹一个 >1 的数；产出慢 → 攒够 1 再弹。

### 4.5 顶部数字

- 左上角两个累计指标：
  - **本日收成**：当地自然日内（午夜重置）所有资产产出之和。
  - **累积收成**：应用使用以来全部产出之和。

## 5. 非目标（MVP 明确不做）

- 不做账号、登录、云同步、多设备。
- 不做真实银行/券商对接、自动拉取行情。
- 不做资产价格波动/亏损模拟（MVP 假设匀速增长）。
- 不做社交、排行榜、分享。
- 不做地图扩张、建造、关卡等重度玩法。

## 6. 成功标准（定性）

- 第一眼"哇，好可爱 / 好治愈"。
- 用户愿意把它挂在桌面，时不时瞥一眼。
- 看着 `+X` 不断冒出来，产生"我的钱在为我工作"的实感。
