# 06 · 关键交互流程

## 1. 首次进入（空地图）

1. **首次启动**先选全局货币（USD / EUR / GBP / JPY / CNY / SGD），并可选开启隐私模式
   （开启则设置全局金币兑换比例）。之后可在设置里改。
2. 展示暖色田园空地图 + 引导卡片："放下你的第一份资产 🌱"。
3. 一个醒目的 `+ 添加资产` 按钮（也常驻于角落）。

## 2. 添加资产向导（多步）

> 核心录入流程，分步降低心智负担；每步可返回。

```
Step 1  选择类型
   ┌──────────┬──────────┐
   │ 现金流    │  投资     │
   │ (年收入)  │ (本金+收益率) │
   └──────────┴──────────┘

Step 2  基本信息
   - 名字（文本）
   - 图标（从预设像素图标网格里选，选中预览运作动画）

Step 3  金额（以全局货币录入；隐私模式下界面提示也会显示对应金币预览）
   - 若现金流：年收入
   - 若投资：当前金额 + 预期年收益率(%)
   （货币与隐私模式是全局设置，不在此步重复选择）

Step 4  落位 & 确认
   - 在地图上选一个空格子放置（或自动找空位，可拖动）
   - 预览：单位 + 进度条开始走动
   - 确认 → 写入 store + 持久化 → 单位正式开始运作
```

校验：每步在"下一步"时做 Zod 校验，错误就地高亮提示（不静默）。

## 3. 货币与隐私模式逻辑（全局）

- 货币与隐私模式都是**全局唯一**设置（见 [02-DATA-MODEL.md](./02-DATA-MODEL.md)）。
- 首次启动时选择货币、可选开启隐私模式（设一个全局金币兑换比例 `goldRate`）。
- 设置页可随时：切换货币、开关隐私模式、改 `goldRate`。
- 切换不重算历史累计（仅影响后续展示）；想要干净的新口径就用「清零按钮」。
- 因为只有一个全局货币与一个全局比例，开启/关闭隐私模式**无需逐个资产补录**。

## 4. 地图与单位交互

- **运作中**：单位循环播放运作动画，头顶进度条爬升。
- **收成瞬间**：满格闪光 + 收成动画 + `+X` 飘字；顶部数字跳动。
- **点击单位** → AssetInspector：查看名称/类型/速率/当日产出；可编辑、删除、移动。
- **拖动单位** → 换格子（可选，MVP 可只在添加时定位）。

## 5. 顶部 HUD

```
┌────────────────────────────────────────────┐
│  ☀ 本日收成  +1,240 ¥        累积收成  +58,300 ¥ │   ← 数字增长时跳动
└────────────────────────────────────────────┘
```

- 隐私模式下单位变为金币图标。
- 本日收成在本地午夜归零（见账本逻辑）。
- HUD 提供一个**清零按钮**（带二次确认）：按下后本日/累积收成归零、重新开始累积，
  不影响资产本身（见 [02-DATA-MODEL.md §5.1](./02-DATA-MODEL.md)）。

## 6. 编辑 / 删除资产

- 编辑金额/收益率/货币/隐私 → 重算 `r` → 重置该资产收成周期（进度条回 0）。
- 删除 → 移除单位精灵；历史累计收成保留（不回滚账本）。

## 7. 离线回归（挂机收益）

- 重新打开应用（或挂件回到前台）时，按离线时长补发收成（见 [03 §5.3](./03-HARVEST-ALGORITHM.md)）。
- 超过阈值合并为一次"离线收成 +N 🌙"汇总提示，避免一次性播放海量动画。

## 8. 空/错误状态

- 无资产：引导添加。
- 校验失败：就地提示。
- 存储失败：toast 报错并保留内存态，提示稍后重试（不静默吞错）。
